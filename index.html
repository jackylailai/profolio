// Side Project · 最近在玩的

Resume Helper.

這是我下班後在做的 side project，主要是想搞清楚「怎麼把 LLM 變成一個能維運的後端模組」——不是 chatbot 那種，是把求職流程切成幾個有邊界的 LLM step，由應用程式控制每一個狀態轉換。LLM 是 bounded provider，不是大腦。

// Architecture

Workflow-first，不是 chatbot-first。

我做這個的初衷是：每一個 LLM 呼叫都應該像呼叫一個一般的後端服務——有明確的 I/O contract、有 schema 驗證、有 prompt 版本、有 cost guardrail、有 regression test。這樣 prompt 改了一行，我也能信任系統行為是可預測的。等於把 LLM 當成「一個會犯錯但邊界清楚」的同事。

Explicit I/O Contracts

每個 LLM call 都寫明 input / output spec，Pydantic schema 卡死，輸出形狀錯了直接 fail-fast。

Prompt Versioning

Prompt 跟 model 版本都進 audit log，能回溯哪個版本產生了哪份輸出，方便 A/B 跟 rollback。

Eval Harness

有 golden fixture 跟 deterministic eval，prompt 改一行就能跑 regression report 看誰變好誰變差。

Cost Guardrails

token / cost / quota / privacy guardrail 內建。預算炸掉之前先擋，每次呼叫都有成本紀錄。

Pluggable Backend

同一份 code 可切 fake / Claude CLI / Anthropic API 三種 backend，離線測試完全不花錢。

App-Owned State

狀態轉換、retry、人工 review gate 全部由 app 控制，不讓 LLM 自己決定下一步。
